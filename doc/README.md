# Documentation Suite

Establish clear communication and maintainable technical lore.

## Skills in this Category
- **[doc-adr-architect](./doc-adr-architect)**: Documenting significant architectural decisions to preserve project context and evolution.
- **[doc-api-auto](./doc-api-auto)**: Professional API documentation strategies using docstrings, OpenAPI, and automated generators.
- **[doc-changelog-keeper](./doc-changelog-keeper)**: Maintaining human-readable, professional version histories using "Keep A Changelog" principles.
- **[doc-inline-clean](./doc-inline-clean)**: Guidelines for writing self-documenting code and knowing when (and how) to use comments.
- **[doc-knowledge-base](./doc-knowledge-base)**: Guidelines for building and maintaining internal project lore, wikis, and onboarding guides.
- **[doc-readme-master](./doc-readme-master)**: Guidelines and templates for crafting high-impact project READMEs that ensure great first impressions.
